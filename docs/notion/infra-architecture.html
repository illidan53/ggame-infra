Platform Relationships

1
GitHub Actions → AWS (OIDC)
GitHub Actions exchanges an OIDC identity token with AWS STS for temporary credentials. No long-lived Access Keys stored anywhere. AWS validates the request via the IAM Role trust policy.
2
GitHub Actions → Pulumi Cloud
Authenticated via PULUMI_ACCESS_TOKEN. Pulumi Cloud stores the state file (current resource state), preventing duplicate resource creation across deployments.
3
Local Dev → AWS (SSO)
Developer logs in via IAM Identity Center through a browser flow, receiving temporary credentials. Used for local pulumi preview during development. Credentials auto-expire.
4
Pulumi → AWS
Pulumi CLI uses the obtained AWS credentials (OIDC or SSO) to call AWS APIs and create/update/delete resources. Infrastructure defined as code, executed as API calls.
